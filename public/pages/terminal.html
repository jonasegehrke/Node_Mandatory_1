Terminal Commands

Description

This page includes all terminal commands that I've personally found helpful specifically for the Node.js elective.
This means that it will not include a lot of basic terminal commands such as cd, ls, mkdir and so on.

Node

Node contains a few commands itself which are useful for your workstation aswell as launching a program.

The command below will tell you which version of node you currently have installed.
node -v

For running a node file once you can use the command below.
node <FILE_TO_RUN>

npm

npm (Node package Manager) is the world's largest Software Registry. Open-source developers use npm to share software in a elegant and easy way
We use npm to install different software such as express.

To install software such as express we use a simple command within our project repository.
npm install express

In some cases we want to install software globally on our device so that we dont have to install it into every repository.
This is done with the -g option.
npm install -g nodemon

nodemon

In the development phase you might want to auto-rerun your program whenever a change is made in your code.
This makes testing much easier to come during the journey

To do this you have to install nodemon with npm.
npm install -g nodemon

After this you can run your program on auto restart mode with the following command.
nodemon.cmd <FILE_TO_RUN>

Heroku

In case you want to publish an express project to the web you can make use of the free hosting service called Heroku
I will explain the path to creating an app with heroku and to push your repository to that app

To install heroku to your device follow this link and download the Heroku CLI

First you have to add a few changes to your express project:

Add a script to start your project in the package.json file

"scripts": {
    "start": "node <FILE_TO_RUN>"
}

Inside your express project change app.listen() to the following:

app.listen(process.env.PORT || 8080, function(){
    console.log("Express server listening on port %d in %s mode", this.address().port, app.settings.env);
});

This will make sure that when deployed to heroku the port will be set by heroku themselves

Now make sure that the updated project is pushed to your git repos.

The next step is to create the heroku app and push the git repos to it.
heroku create
git push heroku main

The project should now get pushed to the app.

In case the project is a subfolder to a git repository use the following command to push instead
git subtree push --prefix <PATH_TO_SUBFOLDER_PROJECT> heroku master

If you need to remove the current heroku app and add a new one you can use the following commands:
git remote rm heroku
heroku git:remote -a <NAME_OF_APP>
